# World Cup 2026 — Correct-Score Predictor ⚽

Predicts the most-likely **scoreline** for every match of the 2026 FIFA World Cup
(all 48 teams, 104 matches), plus advancement odds and a title race, and serves it
through an interactive web app you can host for free.

![tabs](https://img.shields.io/badge/tabs-4-2ea44f) ![teams](https://img.shields.io/badge/teams-48-blue) ![matches](https://img.shields.io/badge/matches-104-orange)

## What it does

There are **two front ends**:

1. **`outputs/dashboard.html`** — a self-contained interactive app (no server, no
   build step). A scrollable, **sortable** (by group or date) master list of all 72
   matches on the left; click any match to open a detail panel on the right with five
   tabs:
   - **Overview** — both predicted scores, win/draw/loss, expected goals, both-teams-to-score, over/under 2.5, **rest hours and travel distance** since each side's previous match, likely scorelines.
   - **The Math** — the full step-by-step derivation of *this* match's score, with a **"why this step" justification** at every stage (see below).
  - **Odds** — **top-5 correct-score** model odds, live bookmaker 1X2 when available, model-vs-live edge, and outright title odds.
   - **History** — head-to-head record and recent form (from `build_history.py`).
   - **Squads** — each side's squad by position (from `data/squads.csv`).

  Open it in any browser, or host it on any static host (GitHub Pages, Netlify, S3…).
  Live bookmaker odds are optional and use the local `odds_proxy.py` helper below.

2. **The Streamlit app (`app.py`)** — the live-tunable version, with sliders for the
   model parameters and a Match Explorer. Deploys to Streamlit Community Cloud.

### The Math tab — how each score is derived

Every match shows the exact arithmetic, with this match's real numbers at each step:
team Elo (plus host boost) → rating gap → goal supremacy → expected goals → injury/
travel/weather/altitude adjustments → Poisson goal distributions → the Dixon-Coles
joint score grid (rendered as a heatmap) → win/draw/loss sums and the final scores.

### Odds, history and squads

- **Odds** are **model-implied** (1 ÷ probability) — fair prices from this model with no
  margin, *not* bookmaker lines. The Odds tab's **Live odds** column pulls real
  correct-score prices from The Odds API through the local `odds_proxy.py` helper,
  which reads `THE_ODDS_API_KEY` from `.env` so the key is not embedded in the
  shareable dashboard HTML. The generated dashboard points at
  `http://127.0.0.1:8787/correct-score?home={home}&away={away}` and refreshes on
  every Odds-tab load. The highest returned live correct-score price among the
  visible top-five correct scores is highlighted when that market is available. If
  the API returns no matching correct-score market, the live correct-score cells are
  left blank and the tab still shows live 1X2 bookmaker prices from the API's `h2h`
  market when available. No bookmaker data is invented.
- **Rest & travel**: each match shows hours since each team's previous game and the
  great-circle distance travelled, computed from the official schedule and venue
  coordinates. Short rest (<96 h) and long travel slightly reduce a team's expected
  goals, and this is **integrated into the odds** (negligible across well-spaced group
  games; larger in fast knockout turnarounds).
- **History** (head-to-head + form) is real data, generated by `build_history.py`.
- **Squads** read from `data/squads.csv`. All degrade to a clear "how to populate"
  message when empty — no players or results are ever invented.

### Two scorelines per match

- **Most likely** — the single most-probable *exact* score (the modal cell of the
  probability grid). For football this is mathematically almost always 1-1 or a
  win-to-nil, because no high-scoring line individually beats them.
- **Expanded** — a livelier model where total goals grow with the mismatch and
  underdogs keep a real attacking threat. It reports the *expected* (average)
  scoreline, so 2-1, 3-1 and 2-2 appear. Same engine, different lens.

### Venues, kickoffs and temperature

Every match is mapped to its real stadium, local date and kickoff from the official
FIFA schedule. Each card shows an expected match-time temperature derived from the
venue's seasonal climate and the kickoff hour — climate-controlled indoor stadiums
(Dallas, Houston, Atlanta) show a steady ~22°C. **Temperatures are seasonal
estimates, not live forecasts** (matches are weeks out).

The sidebar exposes the model parameters (home advantage, goal scale, Dixon-Coles
correction, ML blend, simulation count) so you can tune the model live.

## How the model works

It's a **layered** model, not a black box:

1. **Elo → expected goals.** Each team's current Elo rating is turned into a goal
   *supremacy* and combined with a baseline total to give each side's expected
   goals (λ).
2. **Dixon-Coles correction.** A bivariate adjustment that fixes the well-known
   under-counting of low scores (0-0, 1-0, 0-1, 1-1) — the key to accurate
   *correct-score* prediction. Produces a full score-probability grid; the
   prediction is the grid's most-likely cell.
3. **Adjustment layer.** Multiplicative factors for **injuries**, **travel / rest**,
   **weather** (venue heat + altitude), and **host advantage** (Mexico, USA, Canada
   on home soil). All default to neutral and are fully editable.
4. **ML calibration (optional).** A gradient-boosting model trained on ~150 years of
   international results, blended in by an adjustable weight. **Off by default.**
5. **Monte Carlo simulation.** The full tournament is simulated thousands of times to
   produce advancement and title probabilities.

> **Honesty notes.** Knockout pairings depend on how the groups finish, so knockout
> scores are *conditional* illustrations, not fixed forecasts. Injury and weather
> inputs use neutral defaults until you supply real values. The pure statistical
> engine is the trustworthy default; the ML layer is an enhancement.

## Run it locally

```bash
pip install -r requirements.txt
streamlit run app.py
```

Then open the URL Streamlit prints (usually http://localhost:8501).

To regenerate the CSV predictions and the standalone HTML dashboard:

```bash
python generate_outputs.py 10000      # 10,000 simulations
# -> outputs/predictions_groupstage.csv, tournament_odds.csv, dashboard.html
```

`outputs/dashboard.html` is fully self-contained — open it in any browser, email it,
or drop it on any static host. No server is needed for model odds. For optional live
bookmaker odds, create `.env` with your The Odds API settings and run the local proxy:

```bash
THE_ODDS_API_KEY=your-key-here
THE_ODDS_API_SPORT=soccer_fifa_world_cup
THE_ODDS_API_REGIONS=uk,eu,us,au
THE_ODDS_API_MARKETS=correct_score
```

If Python reports a self-signed certificate chain on a corporate network, prefer
setting `REQUESTS_CA_BUNDLE` to your trusted CA bundle. For a local-only workaround,
set `THE_ODDS_API_VERIFY_SSL=false` in `.env`.

```bash
py odds_proxy.py
```

Then open `outputs/dashboard.html`; the Odds tab will call the proxy on localhost.
If the proxy returns `Invalid markets: correct_score` while standard markets such as
`h2h` work, the API key/sport currently does not have a correct-score market
available. The dashboard will keep the live correct-score column blank, show
model-implied correct-score odds, and still display live 1X2 bookmaker prices when
`h2h` is available.

## Deploy free to Streamlit Community Cloud (≈5 minutes)

1. Push this folder to a **public GitHub repo** (e.g. `worldcup2026-predictor`).
2. Go to **https://share.streamlit.io** and sign in with GitHub.
3. Click **New app**, pick your repo/branch, and set **main file** to `app.py`.
4. Click **Deploy**. Streamlit installs `requirements.txt` automatically and gives
   you a public URL like `https://your-app.streamlit.app`.

That URL is your shareable, always-on front end.

## Populate head-to-head history & squads

Two one-command scripts pull **real** data (run on any machine with internet):

```bash
python build_squads.py      # all 48 final squads     -> data/squads.csv
python build_history.py     # ~47k historical results -> data/history.json
```

Then re-run `python generate_outputs.py` (or reload the Streamlit app): the Squads
tab fills with every team's roster by position, and the History tab shows real
head-to-head records and recent form.

`build_squads.py` ships with Groups A & B (7 verified squads) embedded as an offline
fallback, and downloads the remaining teams from Wikipedia's structured squad data
when run online. Nothing is invented — teams without data show a clear "how to
populate" message instead of placeholder players.

## Keep it live: update predictions as the tournament unfolds

As real games finish, add them to **`data/results.csv`** (one row per match):

```
stage,home_team,away_team,home_score,away_score
group,Mexico,South Africa,2,1
group,South Korea,Czechia,0,0
```

Then re-run `python generate_outputs.py` (or reload the Streamlit app). The model
automatically:

- **Locks in played group games** to their real scores, so qualification and title
  odds now condition on what's actually happened (a team that has won is far more
  likely to advance).
- **Rolls each team's Elo forward** off the real results (winners gain, losers drop,
  weighted by margin), so every *upcoming* match prediction shifts with form.
- **Marks finished matches** with the final score and an **FT** tag in the dashboard,
  and shows whether the model's most-likely call was right.

Use `stage` values `group`, `r32`, `r16`, `qf`, `sf`, `final`. Add results in
chronological order. Team names must match those in `data/elo_ratings.csv`.

### Group-stage motivation layer

The dashboard can also show a separate **Motivation-adjusted read** for unplayed
group-stage fixtures. This is intentionally not used for knockout games, where
every match is already an elimination match. It does **not** change the baseline
predicted score, expanded score, probabilities, backtest checks, advancement odds,
or title odds.

Add explicitly qualified teams to `data/qualified_teams.csv`:

```csv
team,status,note
Argentina,qualified,Already qualified for the knockout stage
France,qualified,Already qualified for the knockout stage
```

Then rebuild the motivation file and dashboard:

```bash
python build_motivation.py      # -> data/motivation_status.csv
python generate_outputs.py 10000
```

For another tournament, update the team/fixture/result data and the explicit
qualified-team file, rerun the same two commands, and the motivation-adjusted score
layer is recalculated from those inputs.

## Validate & calibrate the model (backtest)

Before trusting a forecast model you should measure it. `backtest.py` replays ~47k
historical internationals, rebuilds each team's Elo over time, and scores the model's
probabilities against what actually happened — using the metrics the forecasting
literature uses:

- **RPS** (Ranked Probability Score) — the standard ordered-outcome metric.
- **Log-loss** and **Brier score**, plus accuracy.
- A **reliability diagram** (predicted vs observed) and a **parameter search** that
  re-tunes the hand-set constants on a train split and scores them out-of-sample.

```bash
python backtest.py            # needs internet (downloads the results dataset)
python backtest.py --selftest # offline check of the metric maths
```

Outputs `outputs/backtest_report.txt` and `outputs/reliability.png`. A good football
model sits around **0.18–0.19 RPS**; an uninformed forecast is ~0.22–0.23, so the
report tells you objectively whether the model adds skill, whether it's well
calibrated, and whether re-tuning the constants (or moving to full maximum-likelihood
fitting) is worth it.

## Turn on the ML blend (optional)

```bash
python train_model.py     # downloads match history, trains, writes model.pkl
```

Reload the app — the **ML blend weight** slider activates. Raise it above 0 to mix
the ML goals estimate into the statistical one. On Streamlit Cloud, run this once
locally and commit `model.pkl`, or add it as a build step.

## Update the data

| File | What to edit |
|------|--------------|
| `data/elo_ratings.csv` | Team Elo ratings (refresh from eloratings.net before the tournament). |
| `data/adjustments.csv` | Per-team injury / travel / weather factors (1.0 = neutral). |
| `data/venues.csv` | Per-venue heat index, altitude, timezone offset, seasonal temps, indoor flag. |
| `data/results.csv` | Real match results as they happen — locks played games and updates Elo. |
| `data/fixtures.csv` | Generated — the 72 group matches with venue, local date/kickoff and temperature. |

The official schedule and city→stadium mapping live in `build_data.py`. After
editing teams, venues, or the schedule, run `python build_data.py` to rebuild
`fixtures.csv`, `groups.csv` and `adjustments.csv`.

```

generate_outputs.py builds the interactive dashboard.html + CSV predictions

train_model.py      optional ML calibration layer
data/               Elo, groups, fixtures, venues, adjustments, squads (+ history.json)
outputs/            generated dashboard.html and prediction CSVs
requirements.txt    dependencies
```

*Predictions are probabilistic estimates for entertainment and analysis — not
betting advice.*
